## 1. 旧远程 `main` 最近关键主题

### 1.1 鉴权与权限语义收敛

最近一批提交集中处理了认证失败状态、前端权限语义和误导性权限能力：

- `d7ee1d6` 修复编辑游戏保存流程错误链丢失原因
- `0b7df0d` 重构鉴权失败状态的仓储分层
- `1655f37` 移除存在误导风险的 `v-permission` 伪能力
- `8a41b7e` 收敛前端路由权限语义并修复构建类型错误

可保留结论：

- 编辑保存流程中的错误链路需要明确保留上下文，不应在中途吞掉。
- 认证失败状态应下沉到更稳定的后端仓储/服务边界，不建议继续散落在 handler 或前端兜底。
- 前端的权限表达应尽量收口到路由与明确能力模型，不要继续扩散“看起来能控制权限、实际上只是展示层判断”的伪能力。

### 1.2 移动端与响应式体验优化

旧远程中有一段持续的移动端收敛工作，涉及首页、游戏库、详情页、编辑弹窗、登录页和时间线：

- `ae1cdb6` 收口移动端导航与头部适配
- `65f2a81` 调整时间线加载策略与提示文案
- `2d45190` 调整时间线月份头部与年份轨道间距
- `ee49e15` 精简登录页成功过渡效果
- `144aab6` 优化编辑弹窗媒体区移动端布局
- `17da4f2` 适配编辑游戏弹窗移动端布局
- `2d15834` 优化游戏详情页移动端Wiki展示
- `5a61333` 适配添加游戏弹窗响应式布局
- `4fbfeec` 收口游戏库筛选区响应式适配
- `18bbc90` 优化登录页多端布局与背景层次
- `955a8cf` 优化首页移动端布局与导航样式

可保留结论：

- 移动端适配不是单页问题，而是导航、头部、弹窗、卡片和页面信息密度的整体收敛。
- `App.vue`、`useMenu.ts`、`CardRow.vue`、`GameCard.vue`、`GameCarousel.vue`、详情页与编辑弹窗相关组件是移动端体验的关键触点。
- 首页和书架类页面在移动端应优先保证卡片可滑动、导航可达、信息不过载。

### 1.3 配置与下载链路

旧远程还包含一批偏基础设施的稳定性改进：

- `280a3af` Fix aggregate game metadata updates to use patch semantics
- `fe5d0e8` Refactor frontend asset URL handling
- `fa2c101` 收紧配置解析并显式暴露配置错误
- `fe4a79e` 统一前端下载链路的 API URL 构造
- `71dced9` 优化 SMB 启动脚本交互体验

可保留结论：

- 聚合型元数据更新应使用 patch 语义，避免覆盖式写入导致脏数据。
- 资源 URL 构造和下载链路应统一，不要继续在页面层分散拼接。
- 配置解析失败必须显式暴露错误，而不是静默回退。

## 2. 重建后建议保留的工程记忆

在新仓库继续演进时，建议显式保留以下工程共识：

- 权限控制要基于真实能力边界，避免伪能力和误导性前端指令。
- 移动端适配要覆盖导航、卡片、弹窗、详情页和编辑流程，不只修单页样式。
- 配置、下载、资源 URL、元数据 patch 语义都应保持统一入口。
- 系列库、待处理中心、媒体工作流属于已经被验证过的重要产品方向。
